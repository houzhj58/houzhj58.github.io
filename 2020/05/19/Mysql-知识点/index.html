    1. mysql 索引的实现原理 B+树
      使用B+树实现索引 一般树的深度控制在3-5层
      树的根节点一般常驻内存 根节点大小限制为16KB

    2. 聚簇索引 clustered index 和 非聚簇索引的区别
      innodb实现表的时候 使用聚簇索引
      在innodb中 主键的实现是聚簇索引 非主键索引的叶子节点存储的是主键的值。
      非主键索引 也被称为 二级索引 secondary index
      innodb的存储文件有两个 firm文件和 ibd文件

      myISAM实现表索引的时候 使用非聚簇索引
      myISAM的存储文件有三个 firm myi myd

    3. 如何实现mysql读写分离
      基于主从复制架构 一个主库 挂几个从库 只写主库 主库会自动把数据同步到从库中

      主从复制原理：
      主库将变更写入binlog日志 然后从库将主库的binlog拷贝到自己本地，然后执行binlog中的内容
      以此来保证跟主库的数据保持一致
      主库上的操作可能是并发的 但从库上的操作肯定是串行化的

      如果主库宕机 数据没同步到从库 则可能数据丢失， 为解决这个问题，有两个机制：
      半同步复制- 主机写入binlog后 强制从库同步，从库写入后 会返回一个ack给主库，主库至少收到一个从库
      的ack后 才认为操作是完成的。
      并行复制- 从库开启多个线程 并行读取relaylog中不同库的日志 这是库级别的并行

      主从同步延时问题：
      1.分库 将主库分为多个主库 减小主库的写并发
      2.开启并行复制
      3.重写代码…… 这个……
      4.如果实时性确实要求高的 可以将一部分读操作定位在主库上 （慎重，不推荐）

    4. mysql 事务
      只有使用innodb数据库引擎的数据库或者表 才支持事务
      事务用来维护数据库的完整性 保证成批的sql语句要么全部执行 要不全部不执行

      事务满足的四个条件：ACID Atomicity原子性 Consistency 一致性 Isolation 隔离性 Durability 持久性
